# テスト仕様書

## 概要

カロリー・PFCバランス管理アプリのテスト仕様

## テスト方針

### テストレベル

| レベル     | 説明                                   | 担当   |
| ---------- | -------------------------------------- | ------ |
| 単体テスト | 関数・メソッド単位のテスト             | 開発者 |
| 結合テスト | API・コンポーネント間連携のテスト      | 開発者 |
| E2Eテスト  | ユーザーシナリオに基づく画面操作テスト | QA     |

### テストカバレッジ目標

| 対象                         | 目標    |
| ---------------------------- | ------- |
| バックエンドロジック         | 80%以上 |
| APIエンドポイント            | 100%    |
| フロントエンドコンポーネント | 70%以上 |

### UT-001: ユーザー認証

#### UT-001-001: パスワードハッシュ化

| 項目       | 内容                                         |
| ---------- | -------------------------------------------- |
| テスト対象 | `hashPassword()`                             |
| 目的       | パスワードが正しくハッシュ化されることを確認 |

| No  | テストケース                     | 入力               | 期待結果                |

| 1   | 正常系                           | `"password123"`    | ハッシュ値が生成される  |
